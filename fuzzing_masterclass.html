0. WHAT IS FUZZING? Beginner

  • Definition: Automated process of sending massive amounts of payloads to discover hidden assets or trigger vulnerabilities.
  • Analogy: A robot rapidly trying 50,000 different keys in 10,000 locked doors in seconds.
  • Why it matters: Standard pentesting tests what is visible. Bug bounty requires finding the invisible attack surface.
  • The Mindset: Don't just look for 200 OK. Look for anomalies (weird 403s, unexpected response sizes, or time delays). Anomalies = Vulnerabilities.

1. FUZZING TYPES Intermediate

  • 1. Directory Fuzzing:
    • Goal: Guessing folder names (/admin/, /backup/).
    • Bugs: Exposed panels, source code leaks.
    • Tools: ffuf, gobuster, feroxbuster.
  • 2. Path / File Fuzzing:
    • Goal: Guessing files/extensions (config.php.bak, .env).
    • Bugs: Hardcoded credentials, db backups.
    • Tools: ffuf, dirsearch.
  • 3. Endpoint / API Fuzzing:
    • Goal: Discovering hidden APIs/versions (/api/v2/users).
    • Bugs: BOLA/IDOR, Mass Assignment.
    • Tools: ffuf, kiterunner.
  • 4. Parameter Fuzzing:
    • Goal: Finding hidden inputs (?debug=true, hidden POST JSON keys).
    • Bugs: SSRF, SQLi, Command Injection.
    • Tools: arjun, x8.
  • 5. Subdomain / Vhost Fuzzing:
    • Goal: Brute-forcing Host headers or DNS.
    • Bugs: Internal dev access, subdomain takeover.
    • Tools: ffuf, gobuster.

2. TOOLS DEEP DIVE Advanced

▸ FFUF (Fuzz Faster U Fool)

What & Why: The undisputed king of fuzzing. Extremely fast, written in Go, and highly customizable filtering.

Install 
go install github.com/ffuf/ffuf/v2@latest
FlagMeaning
-wWordlist path (can use multiple: -w w1.txt:FUZZ -w w2.txt:W2)
-uTarget URL (use FUZZ keyword where payload goes)
-mcMatch HTTP Status Codes (e.g., -mc 200,301,302 or -mc all)
-fc / -fsFilter Codes / Filter Size (removes false positives)
-fw / -flFilter by Word count / Line count in response
-acAuto-Calibrate filtering! ffuf automatically detects the "default" response and filters it out. This is the #1 flag most beginners don't know about. Instead of manually figuring out -fs or -fc, just use -ac and ffuf does it for you. Game changer for VHOST and directory fuzzing.
-accCustom auto-calibrate string — use with -acc "keyword" to calibrate on a specific word in the response.
-HAdd custom header (e.g., -H "Cookie: session=123")
-XHTTP method (GET, POST, PUT, DELETE, PATCH)
-dPOST data body (e.g., -d '{"key":"FUZZ"}')
-eComma-separated list of extensions to append (e.g., -e .php,.bak,.env)
-recursionEnable recursive fuzzing (follow discovered directories)
-recursion-depthMax recursion depth (default: 0 = infinite)
-rateRate limit: max requests per second (e.g., -rate 100)
-tNumber of concurrent threads (default: 40)
-timeoutHTTP request timeout in seconds (default: 10)
-xProxy URL (e.g., -x http://127.0.0.1:8080 for Burp/Caido)
-oOutput file (e.g., -o results.json)
-ofOutput format: json, ejson, html, md, csv, all
-requestUse a raw HTTP request file (exported from Burp/Caido)
-request-protoProtocol for raw request (default: https)
-modeMulti-wordlist mode: clusterbomb (all combos) or pitchfork (paired)
-icIgnore comments in wordlist (lines starting with #)
ffuf Examples 
# 1. Basic Directory Fuzz
ffuf -w wordlist.txt -u https://target.com/FUZZ -mc 200,301

# 2. AUTO-CALIBRATE — The most powerful flag! Let ffuf auto-filter noise
ffuf -w wordlist.txt -u https://target.com/FUZZ -ac

# 3. VHOST Fuzzing with -ac (no need to manually find -fs value!)
ffuf -w subdomains.txt -u https://target.com -H "Host: FUZZ.target.com" -ac

# 4. Extension Fuzzing with auto-calibrate
ffuf -w wordlist.txt -u https://target.com/FUZZ -e .php,.bak,.env,.git,.zip -ac

# 5. Fuzzing JSON POST body params with auto-calibrate
ffuf -w params.txt -u https://api.target.com/update -X POST -d '{"FUZZ":"admin"}' -H "Content-Type: application/json" -ac

# 6. Recursive Fuzzing through Burp/Caido Proxy
ffuf -w wordlist.txt -u https://target.com/FUZZ -recursion -recursion-depth 3 -x http://127.0.0.1:8080 -ac

# 7. Rate-limited fuzzing with output to JSON
ffuf -w wordlist.txt -u https://target.com/FUZZ -ac -rate 50 -o results.json -of json

▸ Gobuster & Dirsearch

What & Why: Solid alternatives to ffuf.

# Gobuster (DNS & Dir brute-forcing)
gobuster dir -u https://target.com -w wordlist.txt -t 50

# Dirsearch (Python-based, great default wordlist)
dirsearch -u https://target.com -e php,json,zip -x 400,403,500

▸ Feroxbuster

What & Why: Written in Rust. The absolute best tool for recursive directory busting out of the box.

feroxbuster -u https://target.com -w wordlist.txt --depth 3 --extract-links

▸ Caido & Burp Suite Pro Tip

What & Why: Proxies for interception and targeted fuzzing. Caido is the modern, lightweight Rust alternative to Burp Suite (highly recommended in 2026). Use Caido's "Automate" or Burp's "Intruder" for pinpoint parameter manipulation.

▸ Nuclei

What & Why: Template-based vulnerability scanner, but extremely powerful for fuzzing if you write custom YAML templates to detect specific edge cases.

nuclei -u https://target.com -t fuzzing-templates/ -dast

▸ Arjun & x8

What & Why: Specialized tools for discovering hidden GET/POST parameters.

# Arjun
arjun -u https://api.target.com/v1/user -m GET

# x8 (Rust, very fast)
x8 -u "https://target.com/page?%s=1" -w params.txt

▸ Katana & Hakrawler

What & Why: Crawling tools. You should always crawl to find existing paths BEFORE you start blind fuzzing.

echo "https://target.com" | katana -jc -d 3 | grep "\.js$"

4. ADVANCED FUZZING TECHNIQUES Advanced

Smart Filtering (Beating the WAF & Cloudflare)

If a server returns 200 OK for everything, you must filter. The 2026 meta is using the -ac flag, but if you need manual control: filter by response size (-fs), line count (-fl), or word count (-fw). Get the baseline size of a fake request (e.g., size is 455), then run: ffuf -w wordlist.txt -u http://target.com/FUZZ -fs 455

Fuzzing behind Authentication & WAF Bypass

Pass your session cookie to find privileged endpoints, and bypass WAFs by spoofing the IP.

ffuf -w wordlist.txt -u https://target.com/admin/FUZZ -H "Cookie: session=YOUR_TOKEN" -H "X-Forwarded-For: 127.0.0.1" -ac

Clusterbomb (Multiple Wordlists)

Fuzzing directories and extensions at the same time.

ffuf -w dirs.txt:W1 -w exts.txt:W2 -u https://target.com/W1W2 -mode clusterbomb -ac

REST & GraphQL Endpoint Fuzzing

Found /api/v3/users? Fuzz the version backwards. V1 often lacks security controls. For GraphQL, always fuzz for introspection (__schema) and playgrounds (/graphiql).

# API Versioning Bypass
ffuf -w versions.txt -u https://api.target.com/FUZZ/users

# GraphQL Fuzzing
ffuf -w graphql-endpoints.txt -u https://target.com/FUZZ -mc 200 -ac

5. UNDERRATED TIPS & TECHNIQUES Underrated

  • Underrated Fuzzing 403s (The 2026 Way): If you get a 403 Forbidden on /admin, fuzz bypasses: /admin/, /admin%20/, /%2e/admin, /admin.json, or fuzz headers: -H "X-Custom-IP-Authorization: 127.0.0.1".
  • Underrated FUZZ placement in headers: Don't just fuzz paths. Fuzz the User-Agent, Origin (CORS bypass), and Host headers for Server-Side Request Forgery (SSRF).
  • Underrated Extension Fuzzing: Always append .json to endpoints. /api/users might be 403, but /api/users.json might bypass auth checks in Spring Boot or Ruby on Rails.
  • Underrated S3 Bucket / Cloud Storage Fuzzing: Fuzz bucket names based on company name: company-dev.s3.amazonaws.com, company-assets.s3.amazonaws.com, or Azure blob storage URLs.
  • Underrated JS File Extraction (Source Maps): Crawl for .js.map files. If found, you can recover the entire readable source code of the frontend and extract hidden API endpoints without fuzzing.
  • Underrated Time-Based Fuzzing: Use Burp Suite or Caido to look for responses that take longer than the average. If /?id=1 takes 50ms, but /?id=1' OR SLEEP(5)-- takes 5000ms, you have SQLi.
  • Underrated IIS Short Name Fuzzing: On Windows servers, you can fuzz the 8.3 short names to discover hidden configuration files using tools like IIS-ShortName-Scanner.

6. AI IN FUZZING (2026) Advanced

Modern bug bounty hunters use AI (Claude 3.5+, GPT-4o) to generate context-aware wordlists and automate triage.

Prompt Example for Custom Wordlists:

"I am hunting on a target that uses Next.js, Express, and Kubernetes. The main application is a Fintech trading platform. Generate a list of 100 highly probable hidden API endpoints, administrative paths (e.g., actuator/metrics), and sensitive parameter names specific to this modern 2026 tech stack. AI-Assisted Response Analysis:

Instead of manually parsing thousands of ffuf outputs, pipe the JSON output of ffuf (-o output.json) into a custom script that uses an LLM API to flag anomalous response sizes, weird HTTP headers, or interesting 403 Forbidden patterns that deviate from the norm.

AI + Nuclei Template Generation:

Feed a vulnerable request/response pair to an AI and ask it: "Write a Nuclei YAML template to fuzz for this specific edge case across all subdomains."

7. BUG BOUNTY CHAINING (2026 META) Pro Tip

Finding a path is just step one. You must chain it for impact to get the high payouts.

Chain 1: Directory Fuzzing → Exposed Config → Cloud Takeover

  1. ffuf -w paths.txt -u https://target.com/FUZZ -ac discovers /actuator/env.
  2. Fetch the endpoint to find exposed AWS_ACCESS_KEY_ID and AWS_SECRET_ACCESS_KEY.
  3. Use AWS CLI (or Pacu) to authenticate as the compromised IAM user.
  4. Enumerate permissions, find S3 write access, or escalate to EC2 admin.

Chain 2: Parameter Fuzzing → SSRF → Internal API Access

  1. Find an endpoint that generates a PDF or fetches an image: /api/v1/export.
  2. Use Arjun: arjun -u https://api.target.com/v1/export -m GET.
  3. Arjun discovers a hidden ?url= or ?webhook= parameter.
  4. Craft payload: /api/v1/export?url=http://169.254.169.254/latest/meta-data/iam/security-credentials/.
  5. The generated PDF contains the AWS EC2 instance metadata (High/Critical).

Chain 3: API Version Fuzzing → BOLA (IDOR)

  1. You observe the app using /api/v3/users/123/profile (and modifying it is protected).
  2. Use ffuf to fuzz the version: ffuf -w versions.txt -u https://api.target.com/FUZZ/users/123/profile
  3. You find /api/v1/users/123/profile is still active but lacks authorization checks!
  4. Change the ID to 124 and successfully modify another user's profile (Account Takeover).

Chain 4: Extension Fuzzing → Source Code Leak → RCE

  1. You fuzz extensions on an upload endpoint: ffuf -w exts.txt -u https://target.com/upload/profileFUZZ
  2. You discover profile.php.bak.
  3. Downloading it reveals a hardcoded developer database password.
  4. You log into the exposed phpMyAdmin panel and write a web shell into the web root.

11. HACKER Q&A Beginner

Q: I'm getting 429 Too Many Requests. What do I do?

A: You are hitting a rate limit. Slow down your fuzzing using the -rate flag in ffuf (e.g., -rate 10) and use rotating proxies.

Q: Should I use a massive 10GB wordlist for every target?

A: No. Context is king. A targeted 50k wordlist generated for a specific tech stack (like the ones generated in this masterclass) is far more effective than blindly throwing SecLists at a server.

Q: What is the difference between ffuf and gobuster?

A: ffuf is more versatile—it can fuzz anywhere (headers, JSON bodies, paths). gobuster is primarily designed for directories and DNS, and is sometimes considered slightly more stable for those specific tasks, but lacks the advanced filtering (like -ac) of ffuf.

Q: How do I handle WAFs that block fuzzing?

A: Bypass them by spoofing headers (X-Forwarded-For), using a very low request rate, rotating IPs via tools like fireprox, or finding the origin IP via tools like Censys/Shodan.
